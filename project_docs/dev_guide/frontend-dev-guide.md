### 一、CSS 交互与动画（和你提到的最相关）

| 原则 | 一句话记住 | 解决什么问题 |
|------|-----------|-------------|
| **动画性能黄金法则** | **"动画只动 transform 和 opacity"** | 避免触发重排（reflow）和重绘，防止卡顿。位移用 `translate`，缩放用 `scale`，显隐用 `opacity`。 |
| **层叠上下文原则** | **"z-index 只在同一层叠上下文内比大小"** | 解释了为什么 `z-index: 9999` 有时还是不生效。创建新上下文的方式：`position` 非 static + `z-index`/`opacity`/`transform` 等。 |
| **定位继承原则** | **"绝对定位认最近的 relative 爹"** | `absolute` 的偏移基准是最近的**已定位祖先**（`relative`/`absolute`/`fixed`/`sticky`），不是父元素。 |
| **状态与样式分离** | **"父级管状态（hover/focus），子级管样式（动画/颜色）"** | 就是你分享的原则。用 Tailwind 的 `group` + `group-hover`，或普通 CSS 的 `.parent:hover .child`。 |
| **过渡分离原则** | **"进入和离开的过渡不要写在同一个元素上"** | Vue/React 中常见：进入动画用 `transition-enter`，离开用 `transition-leave`，否则状态切换时动画会互相覆盖。 |

### 二、布局与响应式

| 原则 | 一句话记住 | 解决什么问题 |
|------|-----------|-------------|
| **Gap 优先** | **"用 gap 做间距，不用 margin 做组件间距"** | `gap` 只在 Flex/Grid 子元素之间产生间距，不会和最后一个元素的 margin 冲突，避免 margin collapse 噩梦。 |
| **Flex/Grid 优先** | **"Flex 做一维，Grid 做二维，Float 只用于文字环绕"** | 现代布局不要用 Float 做页面结构，它是为图文混排设计的。 |
| **容器查询思维** | **"组件响应看容器，不是看屏幕"** | 用 `@container`（Container Queries）让组件根据自身宽度调整，而不是全局视口宽度。 |

### 三、组件与状态管理（框架通用）

| 原则 | 一句话记住 | 解决什么问题 |
|------|-----------|-------------|
| **单向数据流** | **"Props 往下流，Events 往上冒"** | 父组件通过 props 传数据，子组件通过事件通知父组件改数据。不要子组件直接改 props。 |
| **展示与逻辑分离** | **"容器组件抓数据，展示组件管样子"** | 一个组件负责获取/处理数据（Smart），一个只负责渲染（Dumb/Presentational）。 |
| **列表渲染原则** | **"列表必加 key，且不要用 index"** | 用唯一 ID 作为 `key`，帮助虚拟 DOM 精准识别元素，避免渲染错乱和性能损耗。 |
| **受控组件原则** | **"表单数据交给 React/Vue 管，不要自己偷偷存"** | 用 state 绑定 `value`，用 `onChange` 更新 state。非受控组件（ref 直接读 DOM）只在特殊场景用。 |
| **状态提升原则** | **"多个组件共享的数据，放到它们最近的共同父级"** | 避免兄弟组件直接通信，减少耦合。 |

### 四、可访问性与健壮性（容易被新手忽略）

| 原则 | 一句话记住 | 解决什么问题 |
|------|-----------|-------------|
| **语义化即免费 A11y** | **"用对 HTML 标签，胜过加十个 ARIA"** | `<button>` 天然支持键盘聚焦和回车触发，`<div onclick>` 不是按钮。先写对标签，再补 ARIA。 |
| **渐进增强** | **"先让内容能用，再让内容好看"** | HTML 结构本身要完整可读，CSS 和 JS 是增强层。关闭 JS 页面也应能看。 |
| **防御性编程** | **"接口返回什么都有可能，不要假设一定对"** | 解构前判断 `null`，数组遍历前判断 `length`，图片加 `onError` 兜底。 |
| **防抖节流原则** | **"高频事件（scroll/resize/input）先节流，再处理"** | 搜索框输入用 `debounce`，滚动监听用 `throttle`，避免性能爆炸。 |

### 五、工程与抽象（关于"什么时候该复用"）

| 原则 | 一句话记住 | 解决什么问题 |
|------|-----------|-------------|
| **WET > DRY** | **"先写两次，再抽象"** | 不要看到两段相似代码就急着封装。过早抽象会导致参数爆炸、耦合加重。第三次重复时再抽组件/函数。 |
| **最小知识原则** | **"组件只接收它需要的数据，不要传整个对象"** | 不要传 `user={user}` 让子组件自己挑，而是传 `userName={user.name}`。减少子组件对外部结构的依赖。 |
