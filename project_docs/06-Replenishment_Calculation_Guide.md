# 补货建议系统说明

> 面向门店管理人员

---

## 核心概念

### 提前期（K）

从下单到货物入库需要的天数。系统会根据每个商品的历史数据自动计算，每个商品的提前期可能不同（0~7天）。

### 当前库存与在途库存

- **当前库存**：今天营业结束后的实际剩余库存
- **在途库存**：已经下单、还在路上、尚未到货的数量（提前期为1天的商品没有在途库存）

### 覆盖天数

```
(当前库存 + 在途库存) ÷ 日均销量 = 还能卖几天
```

系统根据覆盖天数判断库存状态：

| 状态 | 覆盖天数 | 含义 |
|---|---|---|
| 紧缺 | 不到2天 | 需要立即补货 |
| 偏低 | 不到4天 | 建议尽快补货 |
| 正常 | 4~7天 | 库存合理 |
| 充足 | 超过7天 | 暂时不用补货 |

### 建议补货量

```
未来K天的预计需求 - 当前库存 - 在途库存 + 安全库存 = 建议补货量
```

安全库存是为了应对销量波动而额外保留的缓冲量。预测波动越大，安全库存越多。

## 常见问题

**Q：建议补货量是怎么算出来的？**
A：系统会预测这个商品未来几天的销量，减去现有的库存和在途的货，再加上一个安全缓冲量，得出建议补货量。

**Q：为什么有些商品在途库存是0？**
A：有两种情况：一是这个商品的提前期只有1天，昨天下的单今天就到了，没有「在路上」的状态；二是最近没有下过单，自然没有在途。

**Q：显示「数据不足」是什么意思？**
A：这个商品在我们门店的销售记录太少（不到7天），系统没办法做出可靠的预测。建议根据经验判断是否需要补货。

**Q：建议补货量显示0，是不是不用补货？**
A：不一定。0表示按系统预测，现有库存加上在途的货够用。但仍建议关注库存状态，如果临近节假日或有促销活动，可以适当多补。

**Q：安全库存为什么有时候多有时候少？**
A：安全库存和销量波动有关。如果最近销量忽高忽低，系统会加大安全库存来防缺货；如果销量稳定，安全库存就会少一些。

**Q：库存状态显示「充足」，是不是可以不管了？**
A：充足说明近期不会缺货，但仍建议定期查看。库存状态是动态变化的，今天的充足可能过几天就变成正常了。

**Q：系统多久更新一次数据？**
A：每天更新一次，基于前一天结束时的库存和销量数据。

**Q：如果我觉得系统的建议不合理怎么办？**
A：系统建议仅供参考。门店最了解实际情况（如周边活动、天气、节假日等），请结合经验做最终判断。
