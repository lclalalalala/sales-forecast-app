技术架构文档

1. 系统架构

系统采用四层架构，离线计算与在线服务分离。

L1
前端展示层

React + Vite + Recharts，通过 HTTP 调用 API 层获取数据

↕ HTTP GET
L2
API 服务层

Flask REST API，只读查询，不做实时计算

↕ 读取 CSV
L3
数据处理层

离线批处理：构建事实表、维度表、销量汇总、补货推荐

↕ 调用
L4
销量预测层

Walk-forward 组合预测模型，输出未来 7 天逐日预测

运行调用关系

前端 → GET /api/overview Flask API → 读取 CSV 文件
离线 Pipeline → 调用 预测模块 → 输出 CSV 文件

在线 API 不做任何计算，只读取离线预计算的结果。离线 Pipeline 每日运行一次，更新所有 CSV。

2. 数据处理层

离线 Pipeline 按 8 步流程处理原始销售数据，输出预计算表供 API 读取。

2.1 输入

原始文件 sales_data.csv，每行记录一个门店某商品某天的销售快照，包含：日期、门店、商品、类别、开始库存、销量、订单量、结束库存等字段。

2.2 输出表

文件内容
fact_daily_inventory_sales.csv标准化的事实表，每天每门店每商品一行
fact_lead_time.csv每个商品在每个门店的提前期 K 估计值
fact_forecast.csv未来 7 天逐日销量预测
fact_daily_replenishment.csv每日补货推荐（库存 + 预测 + 安全库存 + 建议量）
fact_daily_sales_metrics.csv门店 × 类别维度的日销量汇总

2.3 提前期 K 估计

对每个 (门店, 商品) 组合，用最近 90 天的数据估计 K：

  • 遍历 K = 0~7，用公式 预测明日库存 = 今日库存 - 今日销量 + 第K天前的订单量 还原历史
  • 选择均方误差 (MSE) 最小的 K（误差相同时取较大值）
  • 无法估计时默认 K = 2

2.4 库存递推关系

当日开始库存 = 前日结束库存 + units_ordered(第 K 天前)
当日结束库存 = 当日开始库存 - 当日销量

K=1 时订单隔天到货，在途库存始终为 0；K≥2 时在途库存 = 过去 K-1 天内已下单未到货的订单之和。

3. 销量预测层

3.1 预测模型

采用 Walk-Forward 组合预测，对每个 (门店, 商品) 组合：

  • 基于历史观测数据，训练多种时间序列模型
  • 按日期滚动（walk-forward），每天生成未来 7 天的逐日预测
  • 组合多个模型的预测结果，取加权平均

3.2 预测区间

基于预测误差计算 95% 置信区间：

下界 = 预测值 - 1.96 × σ
上界 = 预测值 + 1.96 × σ

σ 为最近 30 天内、固定 horizon=3 的有效预测误差的标准差。

4. 补货推荐计算

4.1 安全库存

安全库存 = 2.33 × σ × √K
参数含义来源
2.33服务水平系数（对应 99% 不缺货概率）配置
σ预测误差标准差最近 30 天误差统计
K提前期天数Lead Time 估计

4.2 覆盖天数与库存状态

覆盖天数 = (当前库存 + 在途库存) ÷ 日均销量
状态条件
紧缺覆盖 < 2 天
偏低覆盖 < 4 天
正常覆盖 ≤ 7 天
充足覆盖 > 7 天

4.3 建议补货量

建议补货量 = max(0, 未来K天预测需求 - 当前库存 - 在途库存 + 安全库存)

结果向上取整。为 0 时表示现有库存充足，无需补货。
