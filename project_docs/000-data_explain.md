# 数据探索说明文档

## 1. 文档目的

本文档说明库存销售数据的业务范围、数据粒度、字段含义、数据质量及已确认的数据规律。具体的计算公式、预测方法和可配置参数在《计算公式与配置文档》中单独说明。

---

## 2. 数据概览

- **数据用途**：门店销售、库存状态、未来销量预测、补货建议、趋势分析。
- **数据粒度**：`Date + Store ID + Product ID`（日、门店、商品）。
- **数据规模**：`75,755` 行可验证数据，库存关系精确成立，无库存平衡误差。
- **补货滞后 `k`**：按 `(Store, Product)` 组合管理，不是全局固定值。
- **示例数据**：仅 `S001 / P0001 / Electronics / North / 2022-01-01 至 2022-01-16` 用于说明字段结构，不代表完整数据集。

---

## 3. 字段数据字典

### 3.1 标识和分类字段

| 字段 | 类型 | 业务说明 | 当前用途 |
|---|---|---|---|
| `Date` | 日期 | 业务记录对应的日期 | 时间筛选、趋势分析、预测切分 |
| `Store ID` | 字符串 | 门店唯一标识 | 门店筛选和数据隔离 |
| `Product ID` | 字符串 | 商品唯一标识 | 商品筛选、排名和详情 |
| `Category` | 分类 | 商品所属类别 | 类别筛选和分类汇总 |
| `Region` | 分类 | 门店所属区域 | 当前用于数据描述，暂不作为主要筛选维度 |

### 3.2 库存和销售字段

| 字段 | 类型 | 单位 | 业务说明 | 当前用途 |
|---|---|---:|---|---|
| `Inventory Level` | 整数 | 件 | 当天门店商品库存水平，当天销售前的库存水平 | 当前库存、库存趋势、库存风险 |
| `Units Sold` | 整数 | 件 | 当天实际销售数量 | 核心销售指标：分析、排名、预测、补货 |
| `Units Ordered` | 整数 | 件 | 当天订货数量 | 补货关系、在途库存分析 |
| `Demand` | 数值 | 件 | 数据集提供的需求字段 | 当前版本暂不使用 |

**关键说明**：
- `Inventory Level`、`Units Sold`、`Units Ordered` 是三个独立事实指标，不能互相替代。
- 订货数量存在组合级别的滞后 `k`，不等于当天可用库存。
- 库存、销量、订货之间的关系在全部数据上精确成立，具体见计算文档。

### 3.3 价格和促销字段

| 字段 | 类型 | 单位 | 业务说明 | 当前用途 |
|---|---|---:|---|---|
| `Price` | 小数 | 货币 | 商品价格 | 暂待确认口径，暂不作为核心计算字段 |
| `Discount` | 数值 | 百分比 | 折扣比例（如 `5` 表示 `5%`） | 暂不作为核心计算字段 |
| `Promotion` | 0/1 | 无 | 是否促销 | 预测特征或促销分析 |
| `Competitor Pricing` | 小数 | 货币 | 竞争对手价格 | 预测特征或竞争分析 |

### 3.4 外部环境字段

| 字段 | 类型 | 业务说明 | 当前用途 |
|---|---|---|---|
| `Weather Condition` | 分类 | 当天天气 | 后续预测特征 |
| `Seasonality` | 分类 | 当前季节 | 后续预测特征 |
| `Epidemic` | 0/1 | 是否处于疫情影响状态 | 后续预测特征 |

---

## 4. 已确认的数据发现

- 库存关系在全部 `75,755` 行可验证数据上精确成立，无库存平衡误差。
- 补货滞后 `k` 是 `(Store, Product)` 组合级别的属性，需按组合管理。
- `Units Sold` 与 `Demand` 不同，不能互相替代；当前版本预测目标统一为 `Units Sold`。
- `Inventory Level`、`Units Sold`、`Units Ordered` 需依据时间和组合关系共同解释。


---

## 7. 当前业务口径

| 主题 | 当前口径 |
|---|---|
| 销量 | 使用 `Units Sold` |
| 预测目标 | 未来销量 |
| 需求字段 | 暂时忽略 `Demand` |
| 当前库存 | 最新有效日期的 `Inventory Level` |
| 订货数量 | 使用 `Units Ordered` |
| 补货关系 | 按 `(Store ID, Product ID)` 使用对应的 `k` |
| 数据粒度 | 日、门店、商品 |
| 畅销/滞销排序 | 按 `Units Sold` 降序/升序 |
| 库存风险 | 结合库存和未来销量判断 |
| 补货周期 / 库存阈值 | 由配置决定 |
